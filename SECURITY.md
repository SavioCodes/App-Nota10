# Security Policy

## Supported Versions

The `main` branch is considered the supported version.

| Version | Supported |
| ------- | --------- |
| main    | Yes       |
| older branches | No |

## Reporting a Vulnerability

If you discover a security issue, do not open a public issue.

Please report privately by email to: `security@nota10.app`

Include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if available)

Response targets:

- Initial acknowledgement: within 72 hours
- Status update: within 7 days
- Fix or mitigation ETA: as soon as validated

## Scope

This policy includes:

- Backend API (`server/`)
- Authentication and session handling
- Database access and migrations
- Mobile and web clients (`app/`, `lib/`)
- CI/release and repository configuration

## Safe Harbor

We appreciate responsible disclosure and will not pursue legal action for good-faith security research performed under this policy.
